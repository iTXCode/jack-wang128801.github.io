git 学习

版本管理的演变

VCS出现前
  1. 用目录拷贝区别不同版本
  2. 公共文件容易被覆盖
  3. 成员沟通成本高，代码集成效率低下
集中式vcs
  1. 有集中的版本管理服务器
  2. 具备文件版本管理和分支管理能力

git的特点

  1. 最有的存储能力
  2. 非凡的性能
  3. 开源的
  4. 容易做备份
  5. 支持离线操作
  6. 很容易 定制工作流程

安装git

下载地址:https://git-scm.com/book/

git 最小配置

$ git config --global user.name <your_name>

$ git config --global user.email <your@ xx.com>

config的三个作用域

缺省等同于loacl

显示config的配置，加 –list

键Git仓库

两种场景

1.把已有的项目代码纳入Git管理

$cd 项目代码所在的文件夹

$ git init #初始化仓库

2.新建的项目直接用Git管理

$ cd 某个文件夹

$ git init your_project #会在当前路径下创建和项目名称同名的文件夹(生成文件 .git)

git status #查看当前的状态

git log #查看日志

往仓库里添加文件

4次提交，一个像模像样的静态页面生成了
  1. 加入index.html 和 git-logo
  2. 加入 style.css
  3. 加入script.js
  4. 修改index.html 和 style.css

暂存区是为了方便更正

已经添加过的多个文件

git add -u #表示把已经添加过的修改文件一起添加到暂存区

对已经添加到git缓存区的文件改名

1.在本地仓库修改文件名

mv test.c test.md (此处将test.c改成test.md)

2.将已经添加到暂存区的test.c文件删除

git rm test.c

3.再将test.md 添加到缓冲区

git add test.md

*** 删除暂存区工作路径中的所有变更都会被清除的指令

git reset --hard

利用Git命令如何实现呢？(以上三个步骤就会被这一个步骤所替代个)

git mv test.c test.md

git log

查看建立git后的所有commit列表

git log --oneline

查看最近的几个commit记录

git log -nx(查看最近的x次)

git log -nx --oneline (查看最近x次的详细记录)

git log --all (查看所有分支的版本历史)

git log --all --graph(图形化的方式查看分支)

git log --oneline --all -n4 --graph `

查看本地有多少分支

git branch -av(查看所有分支)

git branch -v (查看当前分支)

在网页端查看git的用法

git --help --web log
